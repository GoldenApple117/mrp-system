## 一、快速启动

```bash
# 后端
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload

# 前端（可选，后端自带静态文件）
cd frontend
npm install && npm run dev
```

**访问地址**：前端 `http://127.0.0.1:5173` | 后端 `http://127.0.0.1:8000` | API文档 `http://127.0.0.1:8000/docs`

## 四、认证测试

### 4.1 管理员登录

```bash
# 获取 token
curl -s -X POST http://127.0.0.1:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 返回: { "access_token": "...", "user": {"role":"admin","is_approved":true} }
```

### 4.2 普通用户登录

```bash
curl -s -X POST http://127.0.0.1:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"user1","password":"123456"}'

# 返回: { "access_token": "...", "user": {"role":"normal","is_approved":false} }
```

### 4.3 验证未授权用户被拦截

```bash
TOKEN=$(获取user1的token)
# 业务API → 403
curl -s http://127.0.0.1:8000/api/mps -H "Authorization: Bearer $TOKEN"
# 预期: {"detail":"您的权限尚未通过审批，请联系管理员"}

# 权限API → 正常
curl -s http://127.0.0.1:8000/api/permissions/my-status -H "Authorization: Bearer $TOKEN"
# 预期: {"status":"none","message":"未申请"}
```

## 五、权限流程测试

```bash
ADMIN_TOKEN=$(管理员登录获取token)
USER_TOKEN=$(user1登录获取token)

# 1. user1 申请权限
curl -s -X POST http://127.0.0.1:8000/api/permissions/request \
  -H "Authorization: Bearer $USER_TOKEN"
# 预期: {"message":"申请已提交","status":"pending"}

# 2. admin 查看待审批数量
curl -s http://127.0.0.1:8000/api/permissions/pending-count \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# 预期: {"count":1}

# 3. admin 查看申请列表
curl -s http://127.0.0.1:8000/api/permissions/list \
  -H "Authorization: Bearer $ADMIN_TOKEN"

# 4. admin 同意申请（替换 req_id）
curl -s -X PUT http://127.0.0.1:8000/api/permissions/approve/1 \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# 预期: {"message":"已授权用户 user1","status":"approved"}

# 5. user1 再次访问业务API → 200
curl -s http://127.0.0.1:8000/api/mps \
  -H "Authorization: Bearer $USER_TOKEN"
```

## 九、Railway 云部署测试

### 部署架构

```
Dockerfile 多阶段构建：
  Stage 1 (node:20-alpine) → npm run build → frontend/dist/
  Stage 2 (python:3.11-slim) → pip install → start.sh → uvicorn
```

### 环境变量（生产必设）

- `DB_BACKEND=sqlite`
- `JWT_SECRET_KEY=<随机生成的密钥>`
