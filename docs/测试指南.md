# MRP II 系统测试指南

> 让任何 AI 助手都能快速启动并测试此系统（最新版 v1.3.0）

---

## 一、一键启动

```bash
# 后端
cd mrp-system/backend
python -m pip install -r requirements.txt
python seed_data.py
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000

# 前端
cd mrp-system/frontend
npm install && npm run dev
```

**访问地址：** 前端 `http://127.0.0.1:5173` | API文档 `http://127.0.0.1:8000/docs`

## 二、数据库配置

| 模式 | 配置 |
|------|------|
| **本地开发** | MySQL 8.0（默认）或 `set DB_BACKEND=sqlite` |
| **Railway 部署** | **必须**设置环境变量 `DB_BACKEND=sqlite` |

## 四、Railway 云部署测试

### 4.1 部署架构

```
Dockerfile 多阶段构建：
  Stage 1 (node:20-alpine) → npm run build → frontend/dist/
  Stage 2 (python:3.11-slim) → pip install → start.sh → uvicorn
```

### 4.2 数据持久化（重要）

Railway 部署后需配置 Volume，否则每次部署数据丢失：
1. 项目 → Settings → Volumes → Add Volume
2. 挂载路径填 `/app/backend`
3. 之后数据跨部署持久化

### 4.3 测试流程（部署后）

```bash
# 1. 检查健康状态
curl https://your-domain.up.railway.app/api/health

# 2. 运行 MRP
curl -X POST https://your-domain.up.railway.app/api/mrp/run \
  -H "Content-Type: application/json" -d '{"horizon_days":60}'

# 3. 转换为订单
curl -X POST https://your-domain.up.railway.app/api/mrp/convert-to-orders \
  -H "Content-Type: application/json" \
  -d '{"planned_orders":[...从MRP结果中取...]}'
```

## 五、核心 API 测试

### 5.1 Excel导入（自动建物料）
```bash
curl -X POST http://127.0.0.1:8000/api/bom/import/excel \
  -F "file=@BOM测试-蓝牙音箱.xlsx"
# 预期: 成功导入 1 个BOM，新建物料 15 个
```

### 5.2 BOM版本管理
```bash
curl -X POST http://127.0.0.1:8000/api/bom/version-bump/1 \
  -H "Content-Type: application/json" -d '{"change_reason":"新版"}'
curl -X POST http://127.0.0.1:8000/api/bom/activate/2
```

### 5.3 MRP运算 ⭐
```bash
curl -X POST http://127.0.0.1:8000/api/mrp/run \
  -H "Content-Type: application/json" -d '{"horizon_days":60}'
# 预期: ~74 orders, ~50ms, ~20 exceptions
```

### 5.4 CRP产能计算
```bash
curl -X POST http://127.0.0.1:8000/api/crp/calculate \
  -H "Content-Type: application/json" -d '{"horizon_days":56}'
```

### 5.5 到货检验
```bash
curl -X POST http://127.0.0.1:8000/api/inspection \
  -H "Content-Type: application/json" \
  -d '{"purchase_order_id":1,"item_id":7,"inspect_qty":500}'
curl -X PUT http://127.0.0.1:8000/api/inspection/1/result \
  -H "Content-Type: application/json" -d '{"pass_qty":490,"reject_qty":10}'
```

### 5.6 盘点
```bash
curl -X PUT http://127.0.0.1:8000/api/inspection/stock-counts/1/result \
  -H "Content-Type: application/json" -d '{"actual_qty":195}'
curl -X POST http://127.0.0.1:8000/api/inspection/stock-counts/1/adjust \
  -H "Content-Type: application/json" -d '{"operator":"管理员"}'
```

## 六、报表验证

| 图表 | 数据来源 | 验证方法 |
|------|----------|----------|
| 📊 OTD准时率 | 采购单+工单 → 按周+三色 | 按时/逾期/未完成 |
| 🥧 库存概览 | `/api/inventory/summary` | 按物料类型分 |
| 📈 工单状态 | `/api/production/orders` | 按状态分组 |
| 📋 低库存表 | `/api/inventory/summary` → 过滤 `is_low` | 低于安全库存 |

OTD逻辑：
- 🟢 绿色=按时：状态为"已完成"
- 🔴 红色=逾期：当前日期>需求日期 且 未完成
- ⚪ 灰色=未完成：当前日期<=需求日期 且 未完成
