## 九、前端实现与批量操作

### 9.1 页面功能总览

| 页面 | 批量操作 | 实现方式 |
|------|----------|----------|
| 物料管理 | 多选 → 批量停用 | `@selection-change` + 循环API |
| BOM管理 | Excel导入自动建物料 | 先读物料表→自动创建→再读BOM |
| 库存管理 | 逐条出入库 | 表单提交 |
| MPS计划 | 批量添加(按周自动生成) + 批量删除 | `POST /api/mps/batch` |
| MRP运算 | 一键运算→一键转采购/工单 | 全量计算 |
| 采购管理 | 多选 → 批量下单/批量完成/批量删除 | `selection-change` |
| 生产管理 | 多选 → 批量下达/批量开工/批量完工/批量删除 | `selection-change` |
| CRP产能 | 一键计算 | 基于工单+工艺路线 |
| 检验与盘点 | 逐条创建 | 表单提交 |

### 9.2 批量操作技术实现

所有批量操作使用统一的交互模式：

```vue
<!-- 批量操作栏：选中行后显示 -->
<div v-if="selectedIds.length" class="batch-bar">
  <el-tag>已选 {{ selectedIds.length }} 项</el-tag>
  <el-button @click="batchAction()">批量操作</el-button>
</div>

<!-- 表格开启多选 -->
<el-table @selection-change="rows => selectedIds = rows.map(r => r.id)">
  <el-table-column type="selection" width="50" />
</el-table>
```

```javascript
// 批量删除（通用模式）
async function batchDelete() {
  await ElMessageBox.confirm(`确定删除 ${selectedIds.value.length} 项？`)
  let count = 0
  for (const id of selectedIds.value) {
    try { await api.delete(`/api/items/${id}`); count++ } catch {}
  }
  ElMessage.success(`已删除 ${count} 项`)
  selectedIds.value = []
  fetchData()
}
```

### 9.3 Excel导入增强（重要改进）

导入逻辑已改为自动创建物料：

```
上传Excel文件
  → Step 1: 读取「物料主数据」工作表
    → 遍历每行，查询物料是否存在
    → 不存在则自动创建（material_code唯一约束去重）
  → Step 2: 读取「BOM」工作表
    → 解析父子关系
    → 自动识别顶层物料
    → 递归创建BOM行
  → 返回结果：成功导入X个BOM，新建物料Y个
```

## 十一、部署与运维

### 11.1 本地启动命令

```bash
# 后端
cd mrp-system/backend
python -m pip install -r requirements.txt
python seed_data.py
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000

# 前端
cd mrp-system/frontend
npm install
npm run dev
```

### 11.2 Railway 云部署（Docker 多阶段构建）

**Dockerfile 结构**：
```dockerfile
# Stage 1: 构建前端（node:20-alpine）
FROM node:20-alpine AS frontend
COPY frontend/ . → npm install → npm run build (输出 frontend/dist/)

# Stage 2: Python 后端（python:3.11-slim）
FROM python:3.11-slim
COPY --from=frontend frontend/dist/ ./frontend/dist/
COPY backend/ ./backend/
pip install -r backend/requirements.txt
CMD bash /app/start.sh
```

**关键文件**：
| 文件 | 作用 |
|------|------|
| `Dockerfile` | 多阶段构建（前端 → 后端） |
| `railway.json` | 健康检查配置 `/api/health` |
| `start.sh` | 首次启动自动 seed 演示数据 |
| `.gitignore` | 忽略 `__pycache__`、`node_modules`、`*.db` |

**环境变量**（必须在 Railway Variables 中设置）：
- `DB_BACKEND = sqlite`（必填，否则尝试连接 MySQL 会失败）

**数据持久化**（防止部署丢数据）：
Railway 每次部署会重建容器，SQLite 文件会丢失。配置 Volume 解决：
1. 项目 → Settings → Volumes → Add Volume
2. 名称：`mrp-data`，挂载路径：`/app/backend`
3. Volume 创建后 SQLite 数据跨部署持久化
